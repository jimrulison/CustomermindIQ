2. Banner Management System

The Banner Management system allows administrators to create targeted announcements that appear at the top of user dashboards. This is perfect for training announcements, maintenance notifications, and feature updates.

Creating Announcements

Access Banner Management

In the Admin Panel, click on the "Announcements" tab to view existing banners and create new ones.

Create New Banner

Click "New Announcement" button to open the banner creation form.

Configure Banner Settings

  • Title: Brief, attention-grabbing headline (max 200 characters)
  • Message: Detailed announcement text (max 1000 characters)
  • Type: Info, Warning, Success, Error, Announcement, Training, Maintenance
  • Priority: 0-10 (higher numbers appear first)

Set Targeting Options

  • Target Users: Leave empty for all users, or specify email addresses
  • Target Tiers: Select specific subscription tiers (Starter, Professional, Enterprise)
  • Schedule: Set start and end dates for automatic display control

Advanced Options

  • Dismissible: Allow users to close the banner
  • Call to Action: Optional button text and URL
  • Status: Draft, Active, Scheduled, Paused
üí° Best Practices for Announcements:
  • Use "Training" type for educational content announcements
  • Use "Maintenance" type for scheduled system updates
  • Keep messages concise and action-oriented
  • Target specific tiers for tier-relevant announcements
  • Set appropriate priority levels to avoid banner overload
Managing Existing Banners

View and manage all created banners through the banner management interface:

üìä Banner Analytics

  • View count (how many users saw the banner)
  • Click count (if call-to-action is used)
  • Dismissal count (how many users closed it)
  • Engagement rate calculation

‚öôÔ∏è Banner Actions

  • Edit banner content and settings
  • Toggle active/inactive status

3. Discount Management System

The Discount Management system provides flexible tools for creating and applying discounts to drive user acquisition, retention, and tier upgrades.

Types of Discounts

üí∞ Percentage Discounts

  • Example: 25% off first 3 months
  • Best for: Broad marketing campaigns
  • Value: 1-100 (represents percentage)
  • Use case: General promotions, seasonal sales

üéØ Fixed Amount Discounts

  • Example: $50 off annual subscription
  • Best for: High-value customer retention
  • Value: Dollar amount (e.g., 50.00)
  • Use case: Upgrade incentives, retention offers

üéÅ Free Months

  • Example: 2 months free with annual plan
  • Best for: Annual subscription conversion
  • Value: Number of free months
  • Use case: Trial to paid conversion

üìà Upgrade Discounts

  • Example: 40% off upgrade to Professional
  • Best for: Tier migration campaigns
  • Value: Percentage off upgrade difference
  • Use case: Feature adoption drives
Creating Discount Campaigns

Access Discount Management

In the Admin Panel, navigate to "Settings" ‚Üí "Discount Management" or use the dedicated discount section.

Define Discount Parameters

  • Name: Internal reference name
  • Description: Detailed explanation of the discount
  • Discount Type: Choose from available types
  • Value: Numeric value (percentage, amount, or months)

Set Targeting Rules

  • Target Tiers: Select which subscription tiers are eligible
  • Target Users: Specify individual users by email (optional)
  • Minimum Purchase: Set minimum spending requirement

Configure Usage Limits

  • Usage Limit: Maximum total uses of discount
  • Per User Limit: Maximum uses per individual user
  • Start/End Dates: Schedule discount availability window

Apply Discount to Users

Use the "Apply to User" function to manually assign discounts to specific customers for retention or support purposes.

‚ö†Ô∏è Important Notes:
  • All discount applications are logged for financial auditing
  • Revenue impact is tracked and reported in analytics
  • Expired discounts are automatically deactivated
  • Users can only see discounts they're eligible for

Account Impersonation for Support

Safely impersonate user accounts to provide efficient customer support:

Initiate Impersonation Session

  • Navigate to User Management
  • Find the target user account
  • Click "Impersonate User" button
  • Provide reason for impersonation
  • Set session duration (5 minutes to 8 hours)

Active Impersonation Session

  • Clear indicator showing impersonation mode
  • Access user's view of the platform
  • Ability to navigate all user-accessible features
  • Session timer visible in interface
  • Option to end session early

Security & Audit Features

  • All actions during impersonation are logged
  • Session automatically expires at set time
  • Cannot impersonate other admin accounts
  • Full audit trail for compliance
  • Reason for impersonation recorded
üîí Impersonation Security Guidelines:
  • Only impersonate when necessary for customer support
  • Always provide a clear, detailed reason
  • Use minimum session time required
  • Never modify user data without explicit permission
  • End sessions immediately when support is complete
  • All impersonation activities are audited and logged
